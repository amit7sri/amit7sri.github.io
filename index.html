  • Subdomain Monitoring Code Scanner

    • This is a service that monitors for common security vulnerabilities across twitter subdomains in an automated fashion.Often we get attackers who are able to perform malicious activities on a subdomain of twitter.com, and we often get hacker-one reports about them. There are known vulnerabilities which can be easily automated and hence, improve the company security hygiene. The goal of the system is to find those issues using different available security scanners.This system will have the ability to incorporate different scanning engines that can be configured to be used depending on the needs. The tool gathers the attack surface of twitter.com and its subdomains. Runs security scanner scripts that find specific vulnerabilities and finally reports the results into the dynamic analysis dashboard. The results in the dashboard can then be one-click triaged to JIRA.

  • Subdomain Inventory Framework

    • Since every time we get a HACKER ONE report we need to find information about the subdomain that was attacked, the platform will have the ability provide one single point of lookup. The purpose is to develop a platform wherein information can not only be viewed but also updated for future references as well. For this purpose, I created a subdomain inventory builder which collects the information in an automated manner and the Subdomain Inventory Dashboard which is the platform security's centralized dashboard for displaying information about the subdomains of twitter.com. The dashboard provides the ability to search and update information about the subdomains.
